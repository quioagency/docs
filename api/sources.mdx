A source is the origin of indexed content in Company Brain — it can be a connected integration like Notion or Google Drive, or an upload source you create programmatically to ingest documents directly via the API. The Sources API lets you inspect which sources are active in your workspace, create new upload sources for direct ingestion, and remove sources along with all their associated content. Before filtering search queries by source or uploading documents to a specific bucket, you will need the source IDs that these endpoints return.

## List sources

### GET /api/v1/sources

Retrieve all knowledge sources currently connected to your workspace. Use this endpoint to inspect sync status, check document counts, or fetch source IDs for use with the Knowledge and Search APIs.

#### Example request

```bash
curl https://api.companybrain.ai/v1/sources \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Example response

```json
{
  "data": [
    {
      "id": "src_01HABC",
      "name": "Company Notion Workspace",
      "type": "notion",
      "status": "active",
      "last_synced_at": "2024-01-15T09:00:00Z",
      "document_count": 312
    },
    {
      "id": "src_01HDEF",
      "name": "Engineering Confluence",
      "type": "confluence",
      "status": "syncing",
      "last_synced_at": "2024-01-14T22:00:00Z",
      "document_count": 87
    },
    {
      "id": "src_01HGHI",
      "name": "Direct Uploads",
      "type": "upload",
      "status": "active",
      "last_synced_at": "2024-01-15T11:00:00Z",
      "document_count": 14
    }
  ]
}
```

#### Response fields

<ResponseField name="data" type="array">
  An array of source objects representing every knowledge source in the workspace.

  <Expandable title="Source object fields">
    <ResponseField name="id" type="string">
      The unique identifier for this source. Pass this value as `source_id` in `GET /knowledge` to filter documents, or in the `source_ids` array in `POST /search` to scope queries.
    </ResponseField>

    <ResponseField name="name" type="string">
      A human-readable label for the source, set at connection or creation time.
    </ResponseField>

    <ResponseField name="type" type="string">
      The integration type. Possible values: `notion`, `google_drive`, `slack`, `confluence`, `upload`.
    </ResponseField>

    <ResponseField name="status" type="string">
      The current sync state of the source. Possible values:
      - `active` — the source is connected and its content is up to date.
      - `syncing` — a sync is currently in progress.
      - `error` — the last sync attempt failed. Check the source settings in the Company Brain dashboard for details.
    </ResponseField>

    <ResponseField name="last_synced_at" type="string">
      ISO 8601 timestamp of when the most recent successful sync completed.
    </ResponseField>

    <ResponseField name="document_count" type="integer">
      The number of documents currently indexed from this source.
    </ResponseField>
  </Expandable>
</ResponseField>

## Create a source

### POST /api/v1/sources

Create a new upload source. Upload sources act as named containers for documents you push directly to the knowledge base via `POST /knowledge`. Creating separate upload sources for different teams or content types makes it easier to filter searches and manage content independently.

<Note>
  The Sources API only supports creating `upload` type sources. To connect an integration source such as Notion, Google Drive, Slack, or Confluence, use the **Integrations** section of the Company Brain dashboard. OAuth authorization for third-party tools cannot be completed through the API.
</Note>

#### Body parameters

<ParamField body="name" type="string" required>
  A descriptive name for the new source. Choose something that identifies the content or team it will belong to — for example, `"HR Policy Documents"` or `"Onboarding Materials"`.
</ParamField>

<ParamField body="type" type="string" required>
  Must be `upload`. This is the only source type you can create via the API.
</ParamField>

#### Example request

```bash
curl -X POST https://api.companybrain.ai/v1/sources \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "HR Policy Documents",
    "type": "upload"
  }'
```

#### Example response

```json
{
  "data": {
    "id": "src_01HJKL",
    "name": "HR Policy Documents",
    "type": "upload",
    "status": "active",
    "last_synced_at": null,
    "document_count": 0
  }
}
```

#### Response fields

<ResponseField name="data" type="object">
  The newly created source object.

  <Expandable title="Source object fields">
    <ResponseField name="id" type="string">
      The unique identifier for the new source. Use this as the `source_id` when uploading documents to `POST /knowledge` to associate them with this source.
    </ResponseField>

    <ResponseField name="name" type="string">
      The name you provided in the request body.
    </ResponseField>

    <ResponseField name="type" type="string">
      Always `upload` for sources created through this endpoint.
    </ResponseField>

    <ResponseField name="status" type="string">
      Always `active` immediately after creation.
    </ResponseField>

    <ResponseField name="last_synced_at" type="null">
      `null` for a new source with no documents yet.
    </ResponseField>

    <ResponseField name="document_count" type="integer">
      Always `0` immediately after creation.
    </ResponseField>
  </Expandable>
</ResponseField>

## Delete a source

### DELETE /api/v1/sources/{id}

Remove a source from your workspace. This immediately stops any future syncing and purges all content that was indexed from that source.

#### Path parameters

<ParamField path="id" type="string" required>
  The unique ID of the source to delete. Retrieve this from `GET /sources`.
</ParamField>

#### Example request

```bash
curl -X DELETE https://api.companybrain.ai/v1/sources/src_01HABC \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Response

Returns `204 No Content` on success with an empty response body.

<Warning>
  Deleting a source permanently removes every document indexed from that source from your knowledge base. This cannot be undone. Any search queries that previously returned results from this source will no longer find that content. If you are deleting a connected integration source (such as Notion or Google Drive), the original files in those tools are not affected — only the indexed copies within Company Brain are deleted.
</Warning>
