The Knowledge API gives you full programmatic control over the documents in your workspace. Use it to list everything that has been indexed, push documents directly into the knowledge base without connecting a source, or remove content that is no longer relevant. All requests must include your API key in the `Authorization` header.

## List documents

<br />

### GET /knowledge

Retrieve a paginated list of every document currently indexed in your workspace. Use the optional query parameters to page through large collections or filter results to a single source.

#### Query parameters

<ParamField query="page" type="integer" default="1">
  The page number to retrieve. Pages are 1-indexed.
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Number of documents to return per page. Maximum value is `100`.
</ParamField>

<ParamField query="source_id" type="string">
  Filter results to documents that belong to this source. Omit to return documents from all sources.
</ParamField>

#### Example request

```bash
curl https://api.companybrain.ai/v1/knowledge \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Example response

```json
{
  "data": [
    {
      "id": "doc_01HXYZ",
      "title": "Refund Policy",
      "source_id": "src_01HABC",
      "source_type": "notion",
      "url": "https://notion.so/...",
      "indexed_at": "2024-01-15T10:30:00Z",
      "word_count": 842
    }
  ],
  "meta": {
    "page": 1,
    "per_page": 20,
    "total": 156
  }
}
```

#### Response fields

<ResponseField name="data" type="array">
  An array of document objects matching your query.

  <Expandable title="Document object fields">
    <ResponseField name="id" type="string">
      Unique identifier for the document. Use this ID when deleting a document or referencing it in search results.
    </ResponseField>

    <ResponseField name="title" type="string">
      The document's title as extracted or provided at upload time.
    </ResponseField>

    <ResponseField name="source_id" type="string">
      The ID of the source this document was ingested from. Documents uploaded directly via the API have a `source_type` of `upload` and no `source_id`.
    </ResponseField>

    <ResponseField name="source_type" type="string">
      The integration that produced this document. Possible values: `notion`, `google_drive`, `slack`, `confluence`, `upload`.
    </ResponseField>

    <ResponseField name="url" type="string">
      The original URL of the document in its source system, if available.
    </ResponseField>

    <ResponseField name="indexed_at" type="string">
      ISO 8601 timestamp indicating when the document was last indexed.
    </ResponseField>

    <ResponseField name="word_count" type="integer">
      Approximate number of words in the document's content.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagination metadata for the current response.

  <Expandable title="Meta object fields">
    <ResponseField name="page" type="integer">
      The current page number.
    </ResponseField>

    <ResponseField name="per_page" type="integer">
      The number of results per page used for this response.
    </ResponseField>

    <ResponseField name="total" type="integer">
      Total number of documents across all pages.
    </ResponseField>
  </Expandable>
</ResponseField>

## Upload a document

<br />

### POST /knowledge

Push a document directly into your knowledge base without connecting an external source. This is useful for one-off content, programmatically generated summaries, or migrating content from systems that Company Brain does not yet integrate with.

#### Body parameters

<ParamField body="title" type="string" required>
  A human-readable title for the document. This appears in search results and the knowledge base browser.
</ParamField>

<ParamField body="content" type="string" required>
  The full text of the document. Accepts plain text or Markdown. Company Brain indexes this content and uses it to answer questions.
</ParamField>

<ParamField body="metadata" type="object">
  An optional object of custom key-value pairs to attach to the document. Use metadata to store things like author name, department, or an internal document ID from another system. Keys and values must both be strings.
</ParamField>

#### Example request

```bash
curl -X POST https://api.companybrain.ai/v1/knowledge \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Q4 OKRs",
    "content": "Our Q4 objectives are...",
    "metadata": {
      "author": "Jane Smith",
      "department": "Product"
    }
  }'
```

#### Example response

```json
{
  "data": {
    "id": "doc_01HDEF",
    "title": "Q4 OKRs",
    "source_type": "upload",
    "indexed_at": "2024-01-15T11:00:00Z",
    "word_count": 234
  }
}
```

#### Response fields

<ResponseField name="data" type="object">
  The newly created document object.

  <Expandable title="Document object fields">
    <ResponseField name="id" type="string">
      The unique identifier assigned to your document. Store this if you need to update or delete the document later.
    </ResponseField>

    <ResponseField name="title" type="string">
      The title you provided in the request body.
    </ResponseField>

    <ResponseField name="source_type" type="string">
      Always `upload` for documents created through this endpoint.
    </ResponseField>

    <ResponseField name="indexed_at" type="string">
      ISO 8601 timestamp indicating when indexing completed.
    </ResponseField>

    <ResponseField name="word_count" type="integer">
      Approximate number of words in the content you uploaded.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Indexing is synchronous for uploads — by the time you receive a `201 Created` response, the document is already searchable. Large documents (over 50,000 words) may take a few extra seconds to process.
</Note>

## Delete a document

<br />

### DELETE /knowledge/{id}

Permanently remove a single document from your knowledge base. Once deleted, the document no longer appears in search results or AI-generated answers.

#### Path parameters

<ParamField path="id" type="string" required>
  The unique ID of the document to delete. Retrieve this from the `GET /knowledge` response or from the object returned when the document was uploaded.
</ParamField>

#### Example request

```bash
curl -X DELETE https://api.companybrain.ai/v1/knowledge/doc_01HXYZ \
  -H "Authorization: Bearer YOUR_API_KEY"
```

#### Response

Returns `204 No Content` on success with an empty response body.

<Warning>
  Deleted documents cannot be recovered. If the document originated from a connected source such as Notion or Google Drive, it will be re-indexed the next time that source syncs. To permanently exclude a document from your knowledge base, remove it from the source system before deleting it here, or disconnect the source entirely.
</Warning>
