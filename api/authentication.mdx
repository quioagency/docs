Company Brain uses API keys to authenticate requests to the REST API. Every call you make must include your API key as a Bearer token in the `Authorization` header. If the header is absent or the key is invalid, the API responds with a `401 Unauthorized` error and no data is returned. Keep your keys safe — treat them with the same care as passwords.

## Create an API Key

Generate a key from the Company Brain dashboard. You can create as many keys as you need — for example, one per environment or one per integration.

<Steps>
  <Step title="Log in to the dashboard">
    Open [app.companybrain.ai](https://app.companybrain.ai) and sign in to your workspace.
  </Step>
  <Step title="Open API Keys settings">
    Click your workspace avatar in the top-right corner, then go to **Settings → API Keys**.
  </Step>
  <Step title='Click "Create API Key"'>
    Click the **Create API Key** button in the top-right corner of the API Keys page.
  </Step>
  <Step title="Name your key">
    Enter a descriptive name for the key so you can identify it later — for example, `Production`, `Development`, or `CI Pipeline`. Names don't affect behavior; they're only for your reference.
  </Step>
  <Step title="Copy the key immediately">
    The full API key is displayed **once only** at the moment of creation. Copy it now and store it in a secure location such as a password manager or a secrets manager. After you close the dialog, Company Brain only stores a hashed representation of the key and cannot show it to you again.
  </Step>
</Steps>

<Warning>
  If you lose a key before saving it, you must revoke it and create a new one. There is no way to retrieve a key after the creation dialog is closed.
</Warning>

## Use the API Key

Pass your API key in the `Authorization` header of every request using the `Bearer` scheme:

```bash
curl https://api.companybrain.ai/v1/search \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "What is our refund policy?"}'
```

Replace `YOUR_API_KEY` with the key you copied from the dashboard.

## Code Examples

The examples below show how to configure a persistent HTTP client with your API key so you don't have to set the header on every individual request.

<Tabs>
  <Tab title="Python">
    ```python
    import httpx

    client = httpx.Client(
        base_url="https://api.companybrain.ai/v1",
        headers={"Authorization": "Bearer YOUR_API_KEY"}
    )

    response = client.post("/search", json={"query": "What is our refund policy?"})
    print(response.json())
    ```

    <Tip>
      Replace `YOUR_API_KEY` with `os.environ["COMPANY_BRAIN_API_KEY"]` to avoid hardcoding the key in your source code.
    </Tip>
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://api.companybrain.ai/v1/search', {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ query: 'What is our refund policy?' })
    });

    const data = await response.json();
    console.log(data);
    ```

    <Tip>
      In Node.js, read the key from `process.env.COMPANY_BRAIN_API_KEY` instead of embedding it directly in the source file.
    </Tip>
  </Tab>
  <Tab title="cURL">
    ```bash
    # Store the key in an environment variable first:
    export COMPANY_BRAIN_API_KEY="your_key_here"

    curl https://api.companybrain.ai/v1/search \
      -H "Authorization: Bearer $COMPANY_BRAIN_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"query": "What is our refund policy?"}'
    ```
  </Tab>
</Tabs>

## Security Best Practices

Follow these guidelines to keep your API keys secure:

- **Never commit API keys to source control.** Add secret files and `.env` files to your `.gitignore`. Use a pre-commit hook or a tool like `git-secrets` to catch accidental commits.
- **Use environment variables.** Store the key outside your codebase:
  ```bash
  export COMPANY_BRAIN_API_KEY="your_key_here"
  ```
  Then reference it in code via `os.environ["COMPANY_BRAIN_API_KEY"]` (Python) or `process.env.COMPANY_BRAIN_API_KEY` (Node.js).
- **Create separate keys per environment.** Use a distinct key for development, staging, and production. This way, if a development key is accidentally exposed, your production traffic is not affected.
- **Scope keys by integration.** If multiple services call the Company Brain API, give each its own key. You can revoke a single integration's access without disrupting others.
- **Rotate keys regularly.** Replace production keys on a schedule (e.g., every 90 days) to limit the window of exposure if a key is ever leaked without your knowledge.

## Revoke an API Key

If you suspect a key has been compromised, revoke it immediately:

1. Go to **Settings → API Keys**.
2. Find the key you want to revoke.
3. Click **Revoke** and confirm.

<Warning>
  Revoking a key invalidates it **immediately and permanently**. Any service or integration currently using that key will begin receiving `401 Unauthorized` errors the moment revocation completes. Update all affected services with a replacement key before revoking the old one, or be prepared to update them immediately after.
</Warning>

## Authentication Errors

| HTTP Status | Code | What it means | How to fix it |
|---|---|---|---|
| 401 | `unauthorized` | The `Authorization` header is missing, malformed, or the key does not exist. | Confirm the header is formatted as `Bearer YOUR_API_KEY` and that the key was copied correctly. |
| 403 | `forbidden` | The API key is valid but does not have permission to perform this specific action. | Check whether the key was created with restricted scopes, or contact your workspace admin to verify permissions. |
